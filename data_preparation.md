The dataset consists of labeled cube configurations, where each cube is identified by three visible faces: front, right, and top. The data was curated as a set of six unique cubes, each assigned a triplet of letters (A–C, D–F, G–I, J–L, M–O, P–R). The strings are encoded into numerical tensors, and each input sequence is aligned to match the corresponding output label. The data split is done in a straightforward way, ensuring each cube set is used to train and test the model consistently.
